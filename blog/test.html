Version Control, Git and Github

January 18th, 2015

Version Control is, well, pretty cool. In a nutshell, it helps you keep track of your work, enables you to go back in time, and also facilitates collaboration with others.

Git, along with Subversion, is one of the more popular version control tools. It resides on your local computer, and as you make changes within a git-managed directory, you can check to see that it's monitoring by typing 'git status'. By adding (git add), committing (git commit -m "message"), and then pushing it out to GitHub (git push origin master), it not only records different states (versions) of your work, but it also syncs your work with a central online repository called GitHub.

GitHub is pretty cool too. It's like a central cloud storage system where different users from all over the world are able to sync their local Git repos to the main central GitHub repo. Sort of similar to a DropBox concept, GitHub allows your work to be saved in the cloud, and it allows for multiple users to collaborate on the same software in realtime, by managing changes and making sure all parties are synced up.
